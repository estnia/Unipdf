## 🗂️ 最终项目结构

```
pdfviewer/
├── __init__.py                 # 包导出
│
├── core/                       # 核心领域层
│   ├── document.py             # PDFDocument (357 行)
│   ├── renderer.py             # 渲染函数 (171 行)
│   └── text_engine.py          # 文本引擎 (350 行)
│
├── services/                   # 业务服务层
│   ├── render_service.py       # 渲染服务 (153 行)
│   ├── annotation_service.py   # 注释服务 (315 行)
│   ├── search_service.py       # 搜索服务 (276 行)
│   └── thumbnail_service.py    # 缩略图服务 (113 行)
│
├── workers/                    # 后台工作线程
│   ├── render_worker.py        # 渲染工作线程 (121 行)
│   └── toc_worker.py           # TOC 工作线程 (358 行)
│
├── ui/                         # 界面层
│   ├── annotation_tooltip.py   # 注释提示 (38 行)
│   └── main_window.py          # 主窗口 (906 行) ← 本次迁移
│
└── utils/                      # 工具函数
    ├── patterns.py             # TOC 模式 (77 行)
    └── geometry.py             # 几何工具 (230 行)

主入口:
├── main.py                     # 入口文件 (58 行)

备份:
├── main_original.py            # 原代码备份 (4,137 行)

测试:
├── test_refactoring.py         # 基础测试

```

- Core: 纯数据，无 UI 依赖
- Services: 业务协调
- UI: 界面展示
- Workers: 后台任务

- 先迁移 UI 框架
- 再迁移核心功能
- 最后优化细节
