## 三、各模块功能说明

### 3.1 Core 核心层（`pdfviewer/core/`）

**`document.py`** - 文档模型
- `PDFDocument` 类：封装 `fitz.Document`
- 提供页面管理、文本缓存、元数据访问
- 无 UI 依赖，可在工作线程中使用

**`renderer.py`** - 渲染核心
- `render_page()`：页面渲染
- `render_thumbnail()`：缩略图渲染
- 纯函数设计，不依赖 Qt（除 QImage/QPixmap）

**`text_engine.py`** - 文本引擎
- `TextEngine` 类：文本解析与查询
- 字符级精度处理
- 支持搜索、选择、词组提取

### 3.2 Services 服务层（`pdfviewer/services/`）

**`render_service.py`** - 渲染服务
- 管理 `RenderWorker` 实例
- L2 缓存管理
- 渲染队列调度

**`annotation_service.py`** - 注释服务
- 高亮/下划线注释管理
- 热区索引构建
- 添加/删除注释

**`search_service.py`** - 搜索服务
- 全文档文本搜索
- 结果导航（上/下一个）
- 上下文提取

**`thumbnail_service.py`** - 缩略图服务
- 缩略图生成与缓存
- 按需加载

### 3.3 Workers 工作线程（`pdfviewer/workers/`）

**`render_worker.py`** - 渲染工作线程
- 从原 `main.py` 完整迁移
- 支持视口裁剪渲染（P1 功能）

**`toc_worker.py`** - 目录生成工作线程
- 自动检测文档类型（法律/国标/通用）
- 字体统计分析
- 层级目录构建

### 3.4 Utils 工具层（`pdfviewer/utils/`）

**`patterns.py`** - TOC 模式
- 法律文档正则模式
- GB/T 标准模式
- 通用文档模式

**`geometry.py`** - 几何工具
- 坐标转换（PDF ↔ 屏幕）
- 命中检测
- UI 矩形计算

## 四、架构优势

### 4.1 单一职责原则（SRP）

```
重构前：PDFViewer 类（3,176 行）
  - 窗口管理
  - 渲染逻辑
  - 文本选择
  - 注释管理
  - 搜索功能
  - 全部混在一起！

重构后：
  - core/document.py：仅文档数据
  - services/render_service.py：仅渲染协调
  - services/annotation_service.py：仅注释管理
  - ui/main_window.py：仅窗口框架
```

### 4.2 依赖关系清晰

```
main.py
  └─ ui.main_window
       ├─ services.render_service
       │    └─ workers.render_worker
       │         └─ core.renderer
       ├─ services.annotation_service
       ├─ services.search_service
       └─ services.thumbnail_service
            └─ core.document
```

### 4.3 可测试性提升

- Core 层无 Qt 依赖，可独立单元测试
- Services 层可 Mock 测试
- 各模块可独立验证

## 五、风险控制措施

### 5.1 迁移策略

采用 **"复制 → 替换 → 删除"** 策略：

1. 从 `main.py` 复制代码到新模块
2. 修改导入语句
3. 新 `main.py` 调用新模块
4. 原 `main.py` 备份为 `main_original.py`
5. 测试验证通过后，逐步删除旧代码

### 5.2 向后兼容

- 原代码完整保留为 `main_original.py`
- 可随时切换回旧版本
- 新 `main.py` 仅 50 行，风险极低
